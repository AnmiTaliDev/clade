# Internationalization

Clade is built to be fully translatable. The current default (and only) locale is English. Adding a new language requires creating one JSON file and registering it — no code changes needed elsewhere.

## How it works

All user-visible strings go through the `t()` function from `src/i18n/index.ts`:

```ts
import { t } from "@/i18n";

t("nav.home")                      // → "Home"
t("search.no_results", { query: "XYZ" })  // → "No haplogroups found for "XYZ"."
```

The translation keys live in `src/i18n/locales/en.json`. Every key used in components must exist in that file; missing keys log a warning and return the key string as a fallback.

## Adding a new language

### 1. Create the locale file

Copy `src/i18n/locales/en.json` to `src/i18n/locales/de.json` (or whatever ISO 639-1 code).

Translate every value. Keep all keys identical to the English file — the key structure must not change.

```json
{
  "site": {
    "name": "Clade",
    "tagline": "Die Haplogruppen-Enzyklopädie",
    ...
  }
}
```

### 2. Register the locale

Open `src/i18n/index.ts` and add the import and entry:

```ts
import de from "./locales/de.json";

export const locales = {
  en,
  de,   // add here
} as const;
```

### 3. Add routing (when multi-locale URLs are needed)

Currently there is no locale prefix in URLs (`/` not `/en/`). When adding a second language, you'll need to update `astro.config.mjs` to use Astro's i18n routing and wrap pages accordingly. This is deferred until a second locale is contributed.

## String format

Strings may contain `{param}` placeholders:

```json
"no_results": "No haplogroups found for \"{query}\"."
```

Pass params as the second argument to `t()`:

```ts
t("search.no_results", { query: userInput })
```

Multiple params are supported:

```json
"percentage": "{value}%"
```

## What is NOT translated

- Haplogroup descriptions, interesting facts, and references are stored in the haplogroup data files and are not currently run through the i18n system. They are always in English. Translating haplogroup content is a larger undertaking and is tracked separately.
- Scientific notation and numeric formatting follow English conventions.
